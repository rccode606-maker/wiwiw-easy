# Tiwar Auto-Hunt Bot — запуск на GitHub Actions

Этот проект держит headless-браузер (Playwright + Chromium) открытым на
странице игры и внедряет туда ваш userscript авто-охоты — так он работает
24/7 без необходимости держать ваш ПК включённым и без сна системы.

## Шаг 1. Создайте репозиторий

1. Зайдите на github.com → New repository.
2. Назовите как хотите, например `tiwar-bot`.
3. Поставьте **Private** (рекомендуется — внутри будут cookies от вашего аккаунта).
4. Создайте пустой репозиторий (без README, без .gitignore — они уже есть здесь).

## Шаг 2. Загрузите файлы

В новом репозитории нажмите **Add file → Upload files** и закиньте туда
всю эту папку целиком (включая папку `.github/workflows/` с файлом
`keep-alive.yml` внутри — GitHub Actions подхватит её автоматически).

Файлы, которые должны оказаться в репозитории:
```
.github/workflows/keep-alive.yml
bot.js
package.json
userscript.js
.gitignore
README.md
```

## Шаг 3. Получите cookies своей игровой сессии

Боту нужны cookies, чтобы он зашёл в игру под вашим аккаунтом (логин/пароль
ему передавать не нужно).

1. Откройте tiwar.ru в обычном Chrome, залогинившись как обычно.
2. Откройте DevTools (F12) → вкладка **Application** (или "Приложение") →
   слева **Cookies** → `https://tiwar.ru`.
3. Там будет список cookies (`session`, `auth`, и т.п. — конкретные имена
   зависят от игры). Нужно собрать их в JSON-массив вот такого формата:

```json
[
  {
    "name": "ИМЯ_COOKIE",
    "value": "ЗНАЧЕНИЕ",
    "domain": "tiwar.ru",
    "path": "/",
    "expires": -1,
    "httpOnly": true,
    "secure": true,
    "sameSite": "Lax"
  }
]
```

Проще всего сделать это так:
1. Установите расширение **Cookie-Editor** (есть в Chrome Web Store,
   бесплатное, без рекламы — ищите "Cookie-Editor" от cookie-editor.com).
2. Откройте tiwar.ru, кликните на иконку расширения.
3. Там будет кнопка **Export** → **Export as JSON**. Скопируйте весь JSON.

Это и есть готовый `COOKIES_JSON` — никаких ручных правок делать не нужно.

## Шаг 4. Добавьте секрет в репозиторий

1. В репозитории откройте **Settings → Secrets and variables → Actions**.
2. **New repository secret**.
3. Name: `COOKIES_JSON`
4. Secret: вставьте весь скопированный JSON-массив cookies.
5. Сохраните.

## Шаг 5. Включите Actions и проверьте вручную

1. Откройте вкладку **Actions** в репозитории.
2. Если GitHub попросит подтвердить включение Actions — подтвердите.
3. Слева выберите workflow **Tiwar Auto-Hunt Bot**.
4. Нажмите **Run workflow** (кнопка справа) — это запустит бота прямо сейчас,
   не дожидаясь расписания.
5. Откройте запущенный job, смотрите логи в реальном времени — там будет
   видно `[bot] ...` и `[page] ...` сообщения. Если видите ошибки про
   cookies — значит формат JSON не подошёл, проверьте Шаг 3.

## Как это работает дальше

- Workflow запускается автоматически **каждые 6 часов** по cron
  (`0 */6 * * *`).
- Каждый запуск держит браузер открытым ~5 часов 40 минут (до жёсткого
  лимита GitHub Actions в 6 часов на бесплатных раннерах), затем сам
  закрывается.
- Следующий запуск по расписанию подхватывает работу дальше. Между
  запусками может быть разрыв в несколько минут — это нормальная
  особенность бесплatного тарифа Actions, не баг.
- Раз в 30 минут внутри одного запуска бот сам перезагружает страницу —
  это просто защита от случайных сетевых сбоев, на работу авто-охоты
  не влияет (все настройки/таймеры скрипта хранятся в localStorage
  страницы и переживают перезагрузку).

## Важные предупреждения

- **Лимиты GitHub Actions.** На бесплатном плане у личных аккаунтов
  обычно 2000 минут/месяц на приватных репозиториях (на публичных —
  безлимитно). Бот, работающий почти без перерыва, легко исчерпает этот
  лимит на приватном репо за несколько дней. Если упрётесь в лимит —
  либо сделайте репозиторий **публичным** (тогда Actions бесплатны без
  ограничения по минутам, но тогда никому не показывайте cookies — они
  и так лежат только в секретах, в публичный код не попадают, это
  безопасно), либо ищите платный VPS, как обсуждали раньше.
- **Риск для аккаунта.** Автоматизация по правилам большинства игр может
  считаться ботоводством и вести к блокировке — это ваш риск, не
  технический вопрос.
- Если cookies протухнут (например, истечёт сессия) — бот не сможет
  залогиниться, понадобится обновить секрет `COOKIES_JSON` заново.
